# Security Policy

## Supported versions

Security fixes are applied to the latest `main` branch.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security problems.

Email the maintainer with:

- A short description of the issue
- Steps to reproduce or a proof of concept
- Impact assessment if known

Use the email on the [GitHub profile](https://github.com/jalel-masmoudi) or open a private security advisory on the repository if enabled.

We aim to acknowledge reports within a few days.

## Safe contribution rules

- Never commit secrets (API keys, OAuth secrets, tokens, `.env` files)
- Never paste production credentials into issues or PRs
- Prefer least-privilege tokens for integration examples
